Full-Read SSRF via JSON Connector to AWS Credential Theft

Critical Intigriti March 2026 SSRF / Cloud

TL;DR

Upgraded a blind SSRF to a full-read SSRF by abusing a JSON data connector. The connector fetches arbitrary URLs and stores parsed responses as queryable dataset rows — making any JSON endpoint fully readable. Exploited to steal AWS IAM temporary credentials from the EC2 metadata service (IMDSv1).

Attack Chain

01
Create a JSON connector stream
Register a new data stream using the JSON No Auth connector (com.domo.connector.json)
02
Point it at internal infrastructure
Configure the stream's jsonPath to fetch from AWS IMDS at 169.254.169.254
03
Execute the stream
Trigger execution — the server-side connector fetches the URL and stores the response
04
Query the stolen data
Run SQL against the dataset to read the full response — AWS credentials included

Additional Findings

file:// Protocol Support

The JSON connector also accepts file:// URIs. Testing file:///etc/passwd returned:

ERROR: A JSONObject text must begin with '{' at 1 [character 2 line 1]

The file was read but rejected because /etc/passwd is not valid JSON. Any local JSON configuration file can be fully exfiltrated through this method.

Other Exploitable Connectors

Multiple connector types accept arbitrary URLs and could be weaponized the same way:

  • com.domo.connector.csv.advanced — CSV from HTTP/SFTP/FTP
  • com.domo.connector.xml — XML from URLs
  • com.domo.connector.html-table — HTML tables from URLs
  • com.domo.connector.json.advanced — Advanced JSON with custom parsing

Impact

  • Full-read SSRF — any admin user can read responses from arbitrary internal HTTP endpoints
  • AWS credential theft — IMDSv1 accessible, yielding temporary IAM credentials for the production role
  • Local file read — file:// protocol reads local JSON files from the connector's execution environment
  • Internal network reconnaissance — probe internal services by targeting HTTP endpoints across the VPC

Remediation

  • Block requests to link-local (169.254.x.x), private RFC1918, and loopback addresses from all connectors
  • Migrate all EC2 instances to IMDSv2 (token-based access)
  • Disable file:// protocol support in connectors
  • Implement URL allowlisting for connector data sources
